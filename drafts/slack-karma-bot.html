Using Slack's Real Time Messaging API and Python to Build a Karma Bot

Posted by Bob on Sun 25 June 2017 in Tools • 3 min read

We love Slack! But using their Real Time Messaging API you can make it even cooler. I spent some time today to make a Karma bot which is a nice plugin and I learned a thing or two. In this article I shoare the approach I took.

Setup

This exercise is similar to our How to Build a Simple Slack Bot article. You have to create a bot user and get an API_KEY from Slack. You set these up in your .bashrc or env file.

Bot user needs to be defined as ID so you need to retrieve it with this helper script:

$ python3 -m utils.get_botid
Bot ID for 'karmabot' is U5Z6KGX4L

This calls the get_botid.py script in the utils package. I will post an article on modules and packaging next week, but here you can already see it in action.

The same for the core bot code, I put this in the bot folder and put all setup code like constants, API keys, regular expressions, caching and logging in __init__.py, the file that is meant for setup and that turns a directory into a package.

The script

The main.py script drives the script and calls methods from the bot package:

  • it connects to the Real Time Messaging API with SLACK_CLIENT.rtm_connect()

  • each second it checks our Slack for new messages with the helper parse_next_msg (karma.py) which pings the API with SLACK_CLIENT.rtm_read() and parses the response

  • one of my favorite regex methods findall checks each new message for potential karma actions:

    karma_changes = KARMA_ACTION.findall(text)

    where: 

    KARMA_ACTION = re.compile(r'(?:^| )([^\+\- ]+)\s*([\+\-]+)')

    This is a bit complex so break it down: start of message of preceding space -> one or more characters that are not +, - or space -> 0 or more optional spaces -> the voting component = one ore more +'s and/or -'s

  • karma.py's parse_karma_change is then called to parse out giver, receiver and points. Giver and receiver are returned by the API as IDs so I need slack.py's lookup_username which uses USERNAME_CACHE if username has been looked up before, if not it calls the Slack API.

  • Then change_karma (karma.py) is called to increase/decrease the karma and return a message for the bot to post.

  • Lastly post_msg (slack.py) is called to post the message back to the same channel in Slack the original message originated from.

Simple caching with pickle

For caching results I use Counter and for persistence between script runs pickle

__init__.py:

try:
    logging.info('Retrieving karma cache file')
    karmas = pickle.load(open(KARMA_CACHE, "rb"))
except FileNotFoundError:
    logging.info('No cache file starting new Counter object in memory')
    karmas = Counter()

And a neat trick I learned, in main.py I have a try/finally construct to guarantee cashing of the results when the script ends: 

try:
    # ... main script look ...
finally:
    _end_script()


def _end_script():
    logging.info('Script ended, saving karma cache to file')
    pickle.dump(karmas, open(KARMA_CACHE, "wb"))

I found this technique discussed here.

Deploy

As last time I made run.sh script to keep it alive (auto-respawn). I invoke the script once as a nohup background process on my server:

$ nohup ./run.sh &
